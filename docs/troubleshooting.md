# Troubleshooting

## `401 Unauthorized`

Check these first:

- the `Authorization` header is present
- the header format is `Bearer <token>`
- the token was generated by `microhook generate-token`
- the running config matches the token you expect

## `403 Forbidden`

The token is valid but does not allow that action.

Review `auth.tokens[].actions` and confirm the action name matches exactly.

## `404 action not found`

Check these first:

- the action exists in config
- the action is enabled
- the service was restarted after the config change

## `409 Conflict`

The action uses `concurrency_policy: reject` and another run is already in flight.

If concurrent execution is safe for that action, switch it to `allow`. Otherwise, retry later from the caller.

## Action Timed Out

Symptoms:

- run status is `timed_out`
- `timed_out` is `true`
- `error_summary` mentions the configured timeout

What to check:

- the configured timeout is long enough for normal runtime
- the command is not waiting on interactive input
- the action can reach its `cwd`
- the action can find required executables via `PATH`

## Permission Errors

Microhook actions run as the service user.

Check these first:

- the `microhook` service account can access the configured `cwd`
- any target files or sockets are readable or writable by that user
- service-control commands the action invokes are actually allowed for that account

## Wrong Working Directory

If the action depends on relative paths, set `cwd` explicitly.

Without `cwd`, the child process inherits the service process working directory, which is often not what you intended.

## Missing Environment Variables

Microhook does not pass the full parent environment through to actions.

If an action depends on a value, put it in `env` explicitly.

Remember that Microhook always sets:

- `MICROHOOK_RUN_ID`
- `MICROHOOK_ACTION`
- `MICROHOOK_REQUEST_ID`

## Output Looks Truncated

Microhook stores only the last `max_output_bytes` of stdout and stderr.

Increase `max_output_bytes` if the action needs a larger retained tail.

## `validate-config` Fails

Run:

```bash
microhook validate-config -config /etc/microhook/config.yml
```

Common causes:

- missing `storage.path`
- invalid token format
- duplicate action names
- duplicate token names
- invalid timeout strings
- unsupported `concurrency_policy`
- both `command` and `shell_command` set

## Shell Mode Behaves Differently Than Expected

`shell_command` runs under `/bin/sh -c`.

That means quoting, variable expansion, globbing, and shell builtins apply. If that is surprising or brittle, switch to argv `command` mode.

## Useful Commands

```bash
curl http://127.0.0.1:9464/healthz
microhook version
microhook validate-config -config /etc/microhook/config.yml
journalctl -u microhook -n 200
```
